问题：mysql存储emoji表情存储失败

报错：

1
2

org.springframework.orm.jpa.JpaSystemException: could not execute statement; nested exception is org.hibernate.exception.GenericJDBCException: could not execute statement
Caused by: java.sql.SQLException: Incorrect string value: '\xF0\x9F\x90\xB3aa...' for column 'Content' at row 1

原因：utf8是一种变长的编码方式。它使用1~4个字节表示一个符号，根据不同的符号而变化字节长度。其中emoji表情是4个字节，而mysql的utf编码最多3个字节，所以导致了错误。

解决方案：

  • 方案一：将字符集utf8–》utf8mb4。mysql版本需5.5.3以上
  • 方案二：转成base64再存数据库
1
2
3
4

//方案二
//node
let 编码 = Buffer.from(字符串).toString('base64')
let 解码 = Buffer.from(编码后的字符串, 'base64').toString()

mysql修改表、字段、库的字符集：http://fatkun.com/2011/05/mysql-alter-charset.html

SpringBoot打成war包，部署到Tomcat服务器

步骤

  1. 修改pom.xml文件将默认的jar方式改为war。
  2. 排除内置的Tomcat容器（两种方式都可）。
  3. 启动类继承org.springframework.boot.web.servlet.support.SpringBootServletInitializer，实现configure方法。

注意事项

使用外部Tomcat部署访问的时候，application.properties(或者application.yml)中配置的

1
2

server.port=
server.servlet.context-path=

将失效，请使用tomcat的端口，tomcat，webapps下项目名进行访问。

mvn命令打包

  • clean是清除之前的包，-Dmaven.test.skip=true是忽略测试代码
  • jar 方式打包，使用内置Tomcat：mvn clean install -Dmaven.test.skip=true
    运行：java -jar 包名.jar
  • war方式打包，使用外置Tomcat：mvn clean package -Dmaven.test.skip=true
    运行：${Tomcat_home}/bin/目录下执行startup.bat(windows)或者startup.sh(linux)
