css–:before、:after

::before(:before):创建一个伪元素，其将成为匹配选中的元素的第一个子元素。常通过 content 属性来为一个元素添加修饰性的内容。此元素默认为行内元素。

::after (:after) ：创建一个伪元素，作为选中元素的最后一个子元素。通常会配合 content 属性来为该元素添加装饰内容。这个虚拟元素默认是行内元素。

eg:
a::before { 
  content: "«";
  color: blue;
}
a::after { 
  content: "»";
  color: red;
}

::after表示法是在CSS 3中引入的，::符号是用来区分伪类(:)和伪元素(::)的。支持CSS3的浏览器同时也都支持CSS2中引入的表示法:after。

  • CSS 伪类 是添加到选择器的关键字，指定要选择的元素的特殊状态。例如，:hover 可被用于在用户将鼠标悬停在按钮上时改变按钮的颜色。
  • CSS 伪元素是一个附加至选择器末的关键词，允许你对被选择元素的特定部分修改样式。下例中的 ::first-line 伪元素可改变段落首行文字的样式。

参考文章：

  • https://developer.mozilla.org/zh-CN/docs/Web/CSS/::after

关于前端项目中的字体文件

文件体积大， 加载速度就慢。所以优化的目的就是减小文件的体积。

字体文件使用woff2格式，体积相对较小。

从文件中只提取你需要的字，极大程度的减少体积。

参考文章：

  • https://www.zhangxinxu.com/wordpress/2018/07/known-woff2-mime-convert/ 了解woff2
  • http://forsigner.com/web-fontmin/ Web-Fontmin–在线提取你需要的字体，下载软件更好用
  • https://www.fontke.com/tool/convfont/ 字体格式转换

随思

  • 一心一意，专注于一事。全才在某个领域、在做某件事时也是一心一意的。
  • 针对项目来说，在做这个项目的时候就别想着别的项目，专心把当前的吃透，一心一意才不会得不偿失。
  • 方法论就是需要不断思考、改善，才能发挥越来越大的作用。
  • 学习需要多交流，不要藏着掖着。各有各的发展，你需要关注的是你自身，最大限度效率的提升自己，而不是去阻碍他人。
  • 定期清理网盘的资料，资料在精不在多。存那么多不看的资料，缺点如下：容易过时，选择困难，平添焦虑。互联网时代，需要用时再找也不晚，确定好学习路线了再找。

看书学习三步走：

  1. 理解性地过一遍，划重点，并用文档或本子记录下关键的知识点。（可看完一章或一小章时再回顾记下关键点。此时的笔记自己能懂就行，理解为主，笔记只是辅助理解的。）
  2. 笔记整理。过一遍划线的，补充完整笔记，顺便重现代码。（此时的笔记对自己的要求是有授业性。）
  3. 回顾笔记。自己写的东西，最能引起自己的共鸣，也最能让自己理解。（时常重复3，艾宾浩斯遗忘曲线。）

学习的建议：

  • 步骤：理解，记录，总结，提炼，回顾。
  • 贯穿全线的是思考。
  • 通过博客、笔记记录。
